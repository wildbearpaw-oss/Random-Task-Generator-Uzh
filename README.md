## Описание программы
**Random Task Generator** — это GUI‑приложение для генерации случайных задач из списка. Приложение позволяет добавлять новые задачи, фильтровать их по типу (учёба, спорт, работа, отдых), сохранять историю в JSON‑файл и загружать её обратно.

### Возможности:
- ✅ Генерация случайной задачи по нажатию кнопки
- ✅ Отображение истории сгенерированных задач
- ✅ Фильтрация задач по типу (учёба, спорт, работа, отдых)
- ✅ Добавление новых задач с указанием типа
- ✅ Сохранение истории задач в JSON‑файл
- ✅ Загрузка истории задач из JSON‑файла при запуске
- ✅ Валидация вводимых данных (проверка на пустую строку при добавлении задачи)

## Инструкция по запуску

### Требования:
- Python 3.6 или выше
- Установленная библиотека Tkinter (обычно идёт в комплекте с Python)

### Запуск:
1. Откройте терминал или командную строку.
2. Перейдите в директорию проекта:
   ```bash
   cd RandomTaskGenerator
    Запустите приложение:
    bash
    python main.py
Примеры использования
Генерация задачи:
* Нажмите кнопку «Сгенерировать задачу».
* В верхней части окна появится случайная задача с указанием её типа.
* Задача автоматически добавится в историю внизу окна.

## Фильтрация:
* Выберите тип задачи в выпадающем списке «Фильтр по типу» (например, «спорт»).
* Список истории обновится — в нём останутся только задачи выбранного типа.
* Чтобы увидеть все задачи, выберите «Все» в фильтре.
  Добавление новой задачи:
* Введите текст задачи в поле «Новая задача» (например, «Почитать книгу»).
* Выберите тип задачи из выпадающего списка «Тип» (например, «учёба»).
* Нажмите кнопку «Добавить».
* Новая задача появится в истории и будет участвовать в генерации.
  Сохранение и загрузка истории:
* История автоматически сохраняется в файл tasks.json при добавлении новой задачи.
* При следующем запуске приложения история автоматически загружается из tasks.json.

## Валидация данных
Приложение проверяет:
* При добавлении новой задачи поле ввода не должно быть пустым. Если поле пустое, появится сообщение об ошибке.
* Все типы задач выбираются из фиксированного списка (учёба, спорт, работа, отдых), поэтому ошибок ввода по типу не бывает.
